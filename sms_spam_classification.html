SMS Spam Classification

Introduction

The SMS Spam Classification project focuses on building a machine learning model to classify SMS messages as either "spam" or "not spam" (ham). This report details the various stages of the project, from data cleaning and preprocessing to model building and deployment as a web application.

Methodology

Data Cleaning: The initial dataset contained multiple unnecessary columns, which were removed. Additionally, duplicate records were eliminated, resulting in a clean dataset of 5,169 SMS messages.

Exploratory Data Analysis (EDA): Visualizations were created to explore the distribution of the number of characters, words, and sentences in the messages, revealing a class imbalance between "ham" and "spam" messages.

Data Preprocessing: Text preprocessing involved converting text to lowercase, tokenization, removing special characters, stopwords, and punctuation, and stemming using the Porter Stemmer.

Model Building: Text vectorization was performed using TF-IDF, and various classification models were evaluated, with Multinomial Naive Bayes achieving an accuracy of 97.20%.

Deployment: The final model was deployed as a web application using Streamlit, allowing users to input an SMS message and predict whether it is spam or not.

Conclusion

The SMS Spam Classification project successfully developed a machine learning model to classify SMS messages as spam or not spam. The project also provided a user-friendly web application for real-time predictions.
